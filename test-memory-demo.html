The widget is using several systems to maintain conversation memory:

  • ConversationManager: Stores messages locally and syncs with DynamoDB
  • State Tokens: JWT tokens maintain session continuity
  • Message Buffer: Recent messages are included with each request
  • Streaming Context: Conversation history is embedded in the prompt
